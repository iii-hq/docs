title: 'Concepts'
description: 'The core building blocks of iii-powered applications.'
---

iii is built around two primitives: **Functions** and **Triggers**.

A **Function** is an async handler with a string ID. Register it with the engine from any language, and it becomes triggerable from anywhere in the system — other workers, other languages, other machines.

A **Trigger** is what causes a Function to run. It can be explicit (calling `trigger()` from code) or automatic (HTTP request, cron schedule, queue message, state change, and more).
