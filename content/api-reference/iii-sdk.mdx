The iii SDK connects your application to the iii Engine via WebSocket. It provides function registration, trigger binding, remote invocation, stream management, and observability.

<Cards>
  <Card title="Node.js" href="/docs/api-reference/sdk-node">
    TypeScript / JavaScript SDK. Auto-connects on `init()`. Package: `iii-sdk` on npm.
  </Card>
  <Card title="Python" href="/docs/api-reference/sdk-python">
    Async Python SDK. Explicit `connect()`. Package: `iii-sdk` on PyPI (import as `iii`).
  </Card>
  <Card title="Rust" href="/docs/api-reference/sdk-rust">
    Async Rust SDK. Explicit `connect().await`. Crate: `iii-sdk` (lib `iii_sdk`).
  </Card>
</Cards>

## Quick Comparison

| Concept | TypeScript | Python | Rust |
|---|---|---|---|
| Init | `init(url)` | `III(address=url)` | `III::new(url)` |
| Connect | Auto on `init()` | `await iii.connect()` | `iii.connect().await?` |
| Register function | `iii.registerFunction({ id }, handler)` | `iii.register_function(function_id, handler)` | `iii.register_function(id, \|input\| ...)` |
| Register trigger | `iii.registerTrigger({ type, function_id, config })` | `iii.register_trigger(type, function_id, config)` | `iii.register_trigger(type, function_id, config)?` |
| Context | `getContext()` | `get_context()` | `get_context()` |
| Trigger (sync) | `await iii.trigger(id, data)` | `await iii.trigger(function_id, data)` | `iii.trigger(id, data).await?` |
| Trigger (fire-and-forget) | `iii.triggerVoid(id, data)` | `iii.trigger_void(function_id, data)` | `iii.trigger_void(id, data)?` |
| Shutdown | `await iii.shutdown()` | `await iii.shutdown()` | `iii.shutdown_async().await` |

<Tip>
  The default engine address is `ws://localhost:49134` when running locally.
</Tip>
